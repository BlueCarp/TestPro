# 产品需求规格说明书 — 番茄钟计时器客户端

## 1. 项目概述

本产品是一款番茄钟计时器客户端应用，帮助用户实践番茄工作法（Pomodoro Technique）——将工作拆分为 25 分钟的专注时段（番茄钟），间隔以 5 分钟的短休息，每完成 4 个番茄钟后进行一次长休息（15–30 分钟）。产品以计时器为核心，提供工作/休息循环的自动化切换、通知提醒和基础设置功能，帮助用户提升专注力、管理时间、减少疲劳。

产品定位为**个人效率工具**，面向需要专注工作的知识工作者、学生和自由职业者。[推断-待确认] 平台形态为桌面客户端（Windows/macOS/Linux），优先本地使用，不依赖网络连接。

## 2. 目标用户与用户画像

| 画像 | 描述 | 核心诉求 | 使用场景 |
|------|------|---------|---------|
| 知识工作者 | 办公室职员、远程工作者，每天面对大量脑力任务 | 减少打断、保持专注节奏、防止疲劳 | 工作日 9-18 点连续使用，多个番茄钟循环 |
| 学生 | 大学生、考研/考证备考者 | 克服拖延、量化学习时间、劳逸结合 | 自习时段集中使用 2-4 小时 |
| 自由职业者 | 独立开发者、设计师、写作者 | 自我管理、平衡工作与休息 | 全天分散使用，需要灵活调整时长 |

### 验收标准

#### AC-01：工作计时器
- **Given** 用户打开应用，处于待开始状态
- **When** 用户点击「开始」按钮
- **Then** 计时器从 25:00 开始倒计时，每秒刷新显示，按钮变为「暂停」
- **When** 计时器运行中被切换到后台 [推断-待确认：如果有系统托盘]
- **Then** 计时器在后台继续运行，时间准确

#### AC-02：工作→休息自动切换
- **Given** 工作计时器归零（00:00）
- **When** 系统检测到归零
- **Then** 自动播放结束提示音，弹出桌面通知「工作时间结束，休息一下吧！」
- **And** 计时器自动切换为休息阶段（短休息 5:00 或长休息 15:00）
- **And** 用户无需手动操作即可进入休息倒计时

#### AC-03：长休息判定
- **Given** 用户已完成 3 个工作番茄钟，当前为第 4 个番茄钟的休息阶段
- **When** 系统判断番茄钟计数 = 4
- **Then** 自动进入长休息模式，倒计时显示 15:00
- **Given** 用户已完成 1 个工作番茄钟
- **When** 系统判断番茄钟计数 = 1（不是 4 的倍数）
- **Then** 进入短休息模式，倒计时显示 5:00

#### AC-04：暂停/继续
- **Given** 计时器正在运行，剩余时间为 18:32
- **When** 用户点击「暂停」按钮
- **Then** 倒计时停在 18:32，按钮变为「继续」
- **When** 用户再次点击「继续」
- **Then** 从 18:32 继续倒计时
- **When** 应用意外退出（如崩溃或强制关闭）
- **Then** 重新打开时计时器回到该阶段的初始状态（不保留暂停进度）[推断-待确认]

#### AC-05：重置计时器
- **Given** 计时器正在运行，剩余时间为 10:00
- **When** 用户点击「重置」
- **Then** 计时器回到当前阶段的初始时长（如工作计时器回到 25:00），停止倒计时
- **And** 番茄钟完成计数不变

#### AC-06：跳过当前阶段
- **Given** 工作计时器剩余 15:00
- **When** 用户点击「跳过」
- **Then** 弹出确认提示「确定要跳过当前阶段吗？」
- **When** 用户确认
- **Then** 当前阶段直接结束，按照循环规则进入下一阶段（休息或工作），番茄钟计数相应更新

#### AC-07：时长设置
- **Given** 用户在设置页面
- **When** 用户修改工作时长为 30 分钟、短休息为 10 分钟、长休息为 20 分钟，点击保存
- **Then** 新的计时循环按修改后的时长运行
- **Given** 用户输入非法值（如 0 或负数）
- **When** 用户点击保存
- **Then** 显示提示「请输入有效的时长（1–120 分钟）」，设置不生效

#### AC-08：通知提醒
- **Given** 计时器归零
- **When** 系统触发阶段切换
- **Then** 播放声音提示（系统默认通知音），同时弹出桌面通知（显示阶段名称 + 提示文案）
- **When** 用户在系统设置中关闭了应用的通知权限
- **Then** 通知静默失败，计时器仍正常切换阶段（不因通知失败而卡住）

#### AC-09：番茄钟计数显示
- **Given** 用户开始第一个工作番茄钟
- **When** 番茄钟完成计数为 0
- **Then** 界面不显示番茄钟计数或显示「🍅 × 0」
- **When** 第 1 个工作番茄钟完成，进入短休息
- **Then** 界面显示「🍅 × 1」
- **When** 第 4 个工作番茄钟完成，进入长休息
- **Then** 界面显示「🍅 × 4」
- **When** 长休息结束后用户开始新的工作番茄钟
- **Then** 番茄钟计数重置为 0

#### AC-10：通知开关
- **Given** 用户在设置中关闭声音通知
- **When** 阶段切换触发
- **Then** 不播放提示音，但桌面弹窗（如开启）仍正常显示
- **Given** 用户同时关闭声音通知和桌面通知
- **When** 阶段切换触发
- **Then** 计时器静默切换到下一阶段，无任何通知

## 5. 非功能性需求

| 类别 | 需求 | 说明 |
|------|------|------|
| 性能 | 计时精度 ±1 秒/小时 | 倒计时必须准确，不能出现明显漂移 |
| 性能 | 启动时间 < 2 秒 | 应用冷启动到可操作 |
| 性能 | 后台 CPU 占用 < 1% | 计时器在后台时不应消耗大量资源 |
| 可用性 | 键盘快捷键支持 | 提供开始/暂停（空格键）、重置等常用操作的快捷键 |
| 可用性 | 界面大字显示剩余时间 | 主计时器数字在正常视距下清晰可读 |
| 可用性 | 支持多语言 | 默认简体中文，预留英/日文扩展能力 [推断-待确认] |
| 兼容性 | 操作系统支持 | Windows 10+、macOS 12+ [推断-待确认：是否支持 Linux] |
| 可靠性 | 无网络依赖 | 核心计时功能完全离线可用 |

## 8. 术语表

| 术语 | 定义 |
|------|------|
| 番茄钟（Pomodoro） | 一次完整的 25 分钟工作时段 |
| 短休息（Short Break） | 两个番茄钟之间的 5 分钟休息 |
| 长休息（Long Break） | 每完成 4 个番茄钟后的 15 分钟休息 |
| 循环（Cycle） | 完整的 4 个工作番茄钟 + 3 个短休息 + 1 个长休息 |
| 阶段（Phase） | 计时器当前所处的状态：待开始 / 工作中 / 短休息中 / 长休息中 |
| 番茄钟计数（Pomodoro Count） | 当前循环中已完成的工作番茄钟数量（0-4） |
