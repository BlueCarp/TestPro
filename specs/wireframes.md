# 原型与交互设计 — 番茄钟计时器客户端

## 1. 用户旅程地图

### 主旅程：完成一个完整番茄钟循环

```
[打开应用] → [计时器主页] → [点击「开始」] → [工作计时 25:00 倒计时]
    → [计时归零 → 通知 → 自动切换短休息] → [休息计时 5:00 倒计时]
    → [休息归零 → 通知 → 自动切换工作] → [重复 4 轮]
    → [第 4 个工作归零 → 通知 → 自动切换长休息 15:00]
    → [长休息归零 → 通知 → 循环重置 → 准备下一个循环]
```

### 辅助旅程 A：自定义设置

```
[计时器主页] → [点击设置图标⚙] → [设置页面]
    → [修改工作时长 / 短休息时长 / 长休息时长 / 通知偏好]
    → [保存] → [返回计时器主页，新设置生效]
```

### 辅助旅程 B：手动控制

```
[计时器运行中] → [点击「暂停」] → [计时器暂停] → [点击「继续」] → [恢复计时]
[计时器运行中] → [点击「跳过」] → [确认弹窗] → [确认] → [进入下一阶段]
[计时器暂停中] → [点击「重置」] → [回到当前阶段初始时长]
```

## 2. 页面流程

```
                    ┌─────────────┐
         启动应用 → │  计时器主页  │
                    │     /       │
                    └──────┬──────┘
                           │
                    [⚙ 设置图标]
                           │
                           ▼
                    ┌─────────────┐
                    │   设置页面   │
                    │   /settings  │
                    └──────┬──────┘
                           │
                    [← 返回 / 保存]
                           │
                           ▼
                    ┌─────────────┐
                    │  计时器主页  │
                    └─────────────┘
```

应用为单窗口桌面端，两个页面通过设置图标（⚙）和返回箭头切换。无登录、无引导页。

## 3. 页面清单（含状态）

| 页面 | 路由 | 描述 | 状态列表 |
|------|------|------|---------|
| 计时器主页 | / | 番茄钟计时核心界面，显示倒计时、阶段标签、番茄钟计数、控制按钮 | idle（待开始）/ running_work（工作中）/ paused_work（工作暂停）/ running_short_break（短休息中）/ paused_short_break（短休息暂停）/ running_long_break（长休息中）/ paused_long_break（长休息暂停） |
| 设置页面 | /settings | 时长配置和通知偏好设置 | default / validation_error / saved |

**元素清单**：BackButton（← 返回）、PageTitle（设置）、DurationGroup（工作时长、短休息时长、长休息时长 — 各为一个带 +/- 调节的数值输入）、NotificationGroup（声音通知开关、桌面通知开关 — Toggle 组件）、SaveButton（保存设置）、ResetDefaultsButton（恢复默认设置）

#### 时长调节交互

每种时长是一个独立的调节组件：
```
   工作时间    [ - ]   25   [ + ]   分钟
```
- 点击 `[ - ]` 减少 1 分钟（最小 1 分钟）
- 点击 `[ + ]` 增加 1 分钟（最大 120 分钟）
- 也可直接输入数字

#### 状态：validation_error

```
   ⏱ 计时时长
   ┌──────────────────────────────┐
   │ 工作时间             25 分钟  │
   │ 短休息                0 分钟  │  ⚠ 短休息必须在 1-120 分钟之间
   │ 长休息               15 分钟  │
   └──────────────────────────────┘
```

错误行高亮，保存按钮点击后弹出提示，不离开页面。

## 5. 核心交互流

### 5.1 流程：开始工作计时

1. 用户处于 idle 状态，界面显示 25:00，按钮为「开始」
2. 用户点击「开始」
3. 倒计时开始（每秒 -1），按钮变为「暂停」，状态变为 running_work
4. 番茄钟计数显示当前数量
5. **异常**：点击开始后若计时器线程启动失败 → 回到 idle，提示「计时器启动失败，请重试」

### 5.2 流程：暂停/继续

1. 用户点击「暂停」
2. 倒计时冻结，按钮变为「继续」，状态变为 paused_work
3. 用户点击「继续」
4. 从冻结时间继续倒计时，按钮变回「暂停」，状态回到 running_work

### 5.3 流程：工作计时结束 → 自动切换休息

1. 工作计时器 → 00:00
2. 播放提示音（若声音通知开启），弹出桌面通知「工作时间结束，休息一下吧！」
3. 系统判断番茄钟计数：
   - 计数 < 3 → 短休息，计时器自动设为 5:00（或用户自定义），状态变为 running_short_break
   - 计数 = 3 → 长休息（第 4 个番茄钟完成），计时器设为 15:00（或用户自定义），状态变为 running_long_break
4. 番茄钟计数 +1
5. **通知失败处理**：若系统通知权限关闭，通知静默失败，计时器仍正常切换阶段

### 5.4 流程：休息结束 → 自动切换工作

1. 休息计时器 → 00:00
2. 播放提示音，弹出通知「休息结束，开始工作吧！」
3. 系统判断：
   - 如果是长休息结束 → 番茄钟计数重置为 0
   - 如果是短休息结束 → 计数不变
4. 计时器自动设为工作时长，状态变为 running_work
5. 用户无需操作，直接进入工作倒计时

### 5.5 流程：跳过当前阶段

1. 用户点击「跳过」
2. 弹出确认弹窗：
   ```
   ┌─────────────────────────┐
   │  确定要跳过当前阶段吗？    │
   │                         │
   │  [ 取消 ]    [ 确定 ]    │
   └─────────────────────────┘
   ```
3. 用户点击「确定」→ 当前阶段结束
   - 跳过工作 → 番茄钟计数 +1，进入休息
   - 跳过休息 → 进入工作（计数按规则更新）
4. 用户点击「取消」→ 弹窗关闭，计时器继续

### 5.6 流程：修改设置

1. 用户在计时器主页点击⚙ → 进入 /settings
2. 用户调整时长（点击 +/- 或直接输入数字）
3. 用户切换通知开关
4. 用户点击「保存设置」
   - 校验通过 → 保存成功提示（Toast：「设置已保存」），设置生效
   - 校验失败 → 错误项高亮，提示具体错误信息
5. 用户点击「← 返回」→ 回到计时器主页
   - 若有未保存的修改 → 弹出提示「有未保存的修改，确定离开吗？」→ [放弃] / [继续编辑]

### 5.7 流程：键盘快捷键

| 快捷键 | 作用 | 适用状态 |
|--------|------|---------|
| 空格键 | 开始 / 暂停 / 继续 | 全部 |
| R | 重置计时器 | 暂停中 |
| S | 跳过当前阶段 | 运行中或暂停中 |
| Esc | 关闭确认弹窗 / 返回 | 弹窗打开时 |

## 6. 确认弹窗 & Toast 提示

### 确认弹窗（Modal）

两个场景共用一个 Modal 组件：
- 「跳过当前阶段」确认
- 「放弃未保存的修改」确认

```
┌──────────────────────────────────┐
│                                  │
│       [提示文案]                  │
│                                  │
│    [ 次要操作 ]   [ 主要操作 ]    │
│                                  │
└──────────────────────────────────┘
```

- 主操作在右（确定 / 放弃）
- 次操作在左（取消 / 继续编辑）
- 点击遮罩层 → 等同于次操作（取消）

### Toast 提示

- 保存设置成功：「✅ 设置已保存」— 2 秒自动消失
- 恢复默认设置成功：「✅ 已恢复默认设置」— 2 秒自动消失
- 校验失败提示：红色，需用户手动关闭或 3 秒消失

## 7. 响应式断点

| 设备 | 断点 | 布局变化 |
|------|------|---------|
| 桌面端 | > 1024px | 固定窗口 400×600px，居中显示，不可缩放（minimum window size） |
| 小窗口 | ≤ 400px | 最小窗口尺寸 320×480px，计时器数字等比缩小 |

> 注：V1 仅支持桌面端，移动端响应式设计不在本次范围。
